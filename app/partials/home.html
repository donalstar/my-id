Design Proposal

In today’s world, users do not own or control their data. Nevertheless, users’ profile data, ranging from social media credentials and data to address & demographic information, to sensitive personal and financial data, are readily available and accessible from third party vendors. The user does not control what data is made available, nor who can access it, and the user is not compensated in any way for the shared personal data.

Imagine a world where the user owns his own data, controls which verified data attributes are shared, and gets compensated for sharing of those data attributes. In this world, the consumer of the user’s data may then access the user’s fully-authenticated data attributes, with the knowledge that the user has authorized access to the data attributes.

This is a proposal to design a technical framework to enable this scenario, providing the following:

  • A secure, authenticated interface that allows the user to add verified profile data elements, and to control which data elements are accessible, and to what extent.
  • A means for the user to get compensated for access to particular data elements
  • A secure framework to allow user data elements to be stored in a reliable, tamper-free way, by leveraging the Ethereum blockchain
  • An interface that allows consumers of users’ data to make requests for user data/verify against user data and to pay the appropriate fees, using the Ethereum contract infrastructure.
